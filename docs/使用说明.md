## 1. 快速开始

### 1.1 启动系统

系统依赖 **Docker Desktop** 运行 MySQL 数据库和 **Node.js** 运行应用服务。启动步骤如下：

1. 确保 **Docker Desktop** 正在运行。

2. 打开终端，进入项目根目录，依次执行：

   ```
   powershell
   # 第一步：启动 MySQL 数据库
   .\scripts\start-mysql.ps1
   # `第二步：启动应用服务器`
   .\scripts\start-app.ps1
   ```

3. 看到提示 Rule config demo is running at http://0.0.0.0:5173 后，在浏览器中打开 **http://localhost:5173**。

## 4. 规则管理（管理员）

规则管理是系统的核心前置步骤。只有配置好规则、发布版本、绑定学年后，学生才能开始申报。

### 4.1 概念说明

| 概念 | 含义 |
|---|---|
| **规则集** | 一套完整的综测评分规则模板 |
| **规则版本** | 规则集的某一具体版本（如 v2025-2026） |
| **学年规则快照** | 将版本绑定到学年时生成的不可变副本 |
| **规则节点** | 规则树中的一个节点，分"汇总节点"和"规则项"两种 |
| **汇总节点** | 用于组织树结构，可设定汇总方式（求和/取最高/封顶/扣分）和上限分 |
| **规则项** | 具体的加分/扣分项目，可被学生申报 |

### 4.2 操作流程

`
新建规则集 → 创建规则树 → 配置计分/字段/材料/审核/范围 → 发布版本 → 绑定学年快照
`

#### 4.2.1 新建规则集

1. 点击「规则集与版本」标签页。
2. 在「新建规则集」表单中输入名称和说明，点击「创建规则集」。
3. 左侧边栏会出现新建的规则集。

#### 4.2.2 载入默认规则集

系统内置了一套默认规则集。点击顶部操作栏的「载入默认规则集」按钮，可一键创建示例数据，方便快速体验。

#### 4.2.3 创建规则节点

1. 在左侧边栏选中规则集，然后点击「创建节点」标签页。
2. 选择父节点（根节点表示在规则集下创建顶级分类）。
3. 选择节点类型：
   - **汇总节点**：作为分类容器，需设置汇总方式（求和/取最高/封顶/扣分）和上限分
   - **规则项**：具体加分项，需填写编码和名称
4. 勾选「作为学生申报入口」表示该规则项可被学生申报。
5. 点击「创建节点」。

#### 4.2.4 节点配置

在规则树中点击某一节点后，切换到「节点配置」标签页，可配置以下内容：

| 配置模块 | 说明 |
|---|---|
| **计分配置** | 设置分值计算方式（等级制/权重/公式），如 {"levels":[{"name":"一等奖","score":5}]} |
| **申报字段** | 学生申报时需要填写的字段，如竞赛名称、获奖等级、排名 |
| **材料要求** | 要求上传的证明文件（支持 pdf/jpg/png） |
| **审核要求** | 设置审核角色（班委/学院管理员）、是否需要复核、审核说明 |
| **适用范围** | 限定规则项的适用人群（按专业/年级/学生类型/班级） |
| **团体分配规则** | 团队成果的分数分配方式（负责人/成员、前后50%、等额、自定义） |

#### 4.2.5 发布版本

1. 规则配置完成后，在「规则集与版本」标签页的「新建版本」表单中填写版本号和变更说明。
2. 点击「创建版本」。
3. 也可点击顶部操作栏的「发布版本」按钮。

#### 4.2.6 删除节点/规则集

- **删除节点**：选中节点后，在「节点配置」标签页点击「删除选中节点」
- **删除规则集**：选中规则集后，点击顶部操作栏的「删除当前规则集」

## 8. 审核流程（学委 / 学院管理员）

### 8.1 审核操作

1. 切换到「审核」标签页。
2. 使用筛选下拉选择要查看的申报状态（待审核/已通过/已退回/已驳回）。
3. 点击「刷新审核列表」。
4. 点击左侧某条申报记录，右侧会显示申报详情（包括字段内容、证明材料）。
5. 进行审核操作：

| 操作 | 含义 |
|---|---|
| **审核通过** | 该申报记录进入核算环节 |
| **退回修改** | 退回给学生，学生修改后可重新提交 |
| **驳回** | 该申报记录被拒绝，不可再次提交 |

### 8.2 生成申报样例

点击「生成申报样例」按钮，系统会自动创建一批示例申报数据，用于测试审核流程。

## 9. AI 自动审核

系统在人工审核之外，提供**AI 自动审核**功能作为辅助参考。AI 审核与人工审核并行执行，不影响审核流程。

### 9.1 功能概述

AI 自动审核利用**阿里云 DashScope 多模态视觉模型（qwen-vl-max）**，对上传的证明材料进行 OCR（光学字符识别），自动识别图像中出现的姓名，并与提交材料的**学生真实姓名**进行比对，判断材料是否属于该学生。

### 9.2 审核原则

- **AI 审核与人工审核完全并行**：AI 审核结果不绑定审核流程，不影响人工审核决策
- **仅供参考**：无论 AI 审核结论是"通过"还是"不通过"，最终评分均由人工审核决定
- **中文 + 拼音双语言支持**：材料中学生的姓名可能以中文或拼音形式出现（如"喻锦杨"或"Yu Jinyang"），系统均会自动识别
- **多姓名容错**：证书类材料可能包含多个学生姓名（如团队获奖），只要提交材料的学生姓名出现在图像中即可判定为匹配

### ==9.3 查看 AI 审核结果==

**在审核详情中查看（推荐）：**

1. 以**学委**（`committee001`）或**管理员**（`admin001`）身份登录
2. 切换到「审核」标签页
3. 点击左侧某条申报记录
4. 右侧详情区域的「学生上传材料」和「审核记录」之间，会出现一个 **"AI 自动审核结果" 卡片**

卡片中包含以下信息：

| 信息项 | 说明 |
|---|---|
| **综合结论** | 通过/不通过 + 详细匹配原因 |
| **上传学生** | 提交材料的学生真实姓名 |
| **AI 识别姓名** | AI 从材料图像中识别到的所有姓名 |
| **按材料明细** | 每个上传附件（图片/PDF）的逐条 OCR 识别结果和匹配详细说明 |

### 9.4 手动触发 AI 审核

当申报已提交但需要重新进行 AI 审核时（无需重新提交申报），可使用以下方式手动触发：

```powershell
curl -X POST http://localhost:5173/api/ai-audit/trigger/{申报ID}
```

触发后等待几秒，AI 审核结果会自动写入数据库，刷新审核详情页面即可看到最新结果。

### 9.5 AI 审核的工作流程

```
学生提交材料
    ↓
系统异步触发 AI 审核（不阻塞页面响应）
    ↓
从学生档案中获取学生真实姓名
    ↓
将中文姓名转换为多种拼音格式
    ↓
对每份附件调用 DashScope OCR 识别图像文字
    ↓
从 OCR 结果中提取人名（支持中文2-4字、英文/拼音格式）
    ↓
5层逐步匹配策略：
  1. 中文精确匹配（如 "喻锦杨" = "喻锦杨"）
  2. 拼音空格格式（如 "Yu Jin Yang"）
  3. 拼音连写格式（如 "YuJinYang"）
  4. 字符包含匹配（OCR文本包含姓氏和名字各字符）
  5. 拼音包含匹配（OCR英文文本包含拼音片段）
    ↓
写入 ai_audit_record 表（综合结论 + 按附件明细）
    ↓
审核详情页面实时展示 AI 审核结果
```

### 9.6 注意事项

- AI 审核在申报**提交后异步执行**，通常需要几秒到十几秒完成
- 如果材料中不存在学生姓名（如证明材料为纯图表），则"match_success"为0（不通过），这属于正常的识别结果
- 如果材料图像质量较低或为纯文本表格扫描件，识别准确率可能下降
- 当前 AI 模型仅支持图像文件（jpg/png），PDF 文件需要转换为图像后再处理

## 10. 统一上传（学委 / 学院管理员）

此功能用于管理员**批量导入**学生的加分数据，适用于以下场景：

- 学院统一组织的活动（如运动会、志愿服务时长等）
- 无需学生逐个申报的固定加分项

### 10.1 上传步骤

1. 切换到「统一上传」标签页。
2. 选择学年和目标规则项。
3. 点击「下载模板」获取 Excel 模板。
4. 按模板格式填写学生学号和加分信息。
5. 选择填写好的 xlsx 文件。
6. 点击「开始上传」。

上传后每行数据会自动生成一条已批准的申报记录，并触发核算。

> 📌 学委只能上传本行政班学生数据，学院管理员可上传全院学生数据。

## 14. 常见问题

### Q1：为什么登录后看不到某些功能？

不同角色的功能不同。请确认你选择了正确的角色登录。学生只能申报和查看自己的结果；规则管理和学年管理需要学院管理员或最高管理员权限。

### Q2：审核时提示"请先登录后再执行此操作"？

登录会话可能已过期，请刷新页面重新登录。

### Q3：学生申报时找不到可申报项目？

可能原因：
1. 当前学年尚未绑定规则快照（需要管理员在「学年管理」中完成绑定）
2. 规则项没有勾选「作为学生申报入口」
3. 该规则项的适用范围不包含当前学生

### Q4：如何修改已发布版本的规则？

不能直接修改已发布的版本。你需要：
1. 在规则集中创建新版本
2. 在新版本中修改规则
3. 将新版本重新绑定到学年

### Q5：核算结果不正确怎么办？

1. 检查规则配置是否正确（计分配置、汇总方式、上限分等）
2. 检查申报记录的审核状态（只有审核通过才会参与核算）
3. 在「核算」标签页点击「手动重算」触发重新计算

### Q6：数据库连接失败？

1. 确认 Docker Desktop 正在运行
2. 确认已执行 .\scripts\start-mysql.ps1 且显示 "MySQL is ready"
3. 检查 .env 文件中的数据库配置是否正确
