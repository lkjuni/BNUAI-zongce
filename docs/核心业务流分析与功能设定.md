# 学年与规则管理

### 1. 业务背景

规则由学院根据每学年实际情况制定，规则内容，分值标准等变化而调整。例如，某学年新增“人工智能创新竞赛”加分项 、国家级一等奖由 5 分调整为 6 分。
因此，系统需要提供规则管理功能，使管理员能够预设、修改并按学年指定综测规则集，从而支撑申报、审核和核算。

### 3. 核心概念


| 概念         | 含义                                                              |
| ------------ | ----------------------------------------------------------------- |
| 规则集       | 一套完整的综测评分规则，可被某一学年使用                          |
| 规则项分类   | 对规则项进行分类管理，如科研竞赛、志愿服务、学生工作、文体活动等  |
| 规则项       | 具体的加分、扣分或计分项目，如“国家级竞赛一等奖加 5 分”         |
| 类内规则     | 某一分类内部统一适用的限制或计算规则，如“科研竞赛类最高加 8 分” |
| 团体分配规则 | 团队成果在不同成员之间如何分配分数，如队长满分、成员按比例得分    |
| 学年规则绑定 | 为某一学年指定其使用的规则集                                      |
| 规则变更重算 | 当已绑定学年的规则集发生变化时，系统重新计算该学年综测成绩        |
| 可申报范围   | 用户在前端界面可以选择的申报项范围                                |
|              |                                                                   |
|              |                                                                   |

### 4. 规则集结构

同一个子类别中的项目取最高分且不累加，不同子类别的项目可以累加

规则集采用树状结构组织：

```
规则集
├─ 规则项分类
│  ├─ 类内规则
│  ├─ 规则项
│  ├─ 规则项
│  └─ 规则项
├─ 规则项分类
│  ├─ 类内规则
│  ├─ 规则项
│  └─ 规则项
└─ 团体分配规则
```

示例：

```
2024-2025 学年综测规则集
├─ 科研竞赛类
│  ├─ 类内规则：本类最高加 8 分，同一作品取最高奖项
│  ├─ 国家级竞赛一等奖：加 5 分
│  ├─ 省部级竞赛一等奖：加 3 分
│  └─ 校级竞赛一等奖：加 1 分
├─ 志愿服务类
│  ├─ 类内规则：最高加 3 分
│  └─ 志愿服务时长：每 10 小时加 0.5 分
└─ 团体分配规则
   └─ 队长按 100% 计分，队员按 80% 计分
```

### 5. 业务流程

规则管理的核心业务流程如下：

```
管理员新建或复制规则集
        ↓
配置规则项分类、类内规则、规则项、团体分配规则
        ↓
为指定学年绑定规则集
        ↓
学生按照规则集进行申报
        ↓
审核人员依据规则集进行材料审核
        ↓
系统根据规则集计算综测成绩
        ↓
若规则集发生修改，系统提示影响范围并触发成绩重新计算
```

### 6. 功能需求


| 功能             | 说明                                                                     |
| ---------------- | ------------------------------------------------------------------------ |
| 新建规则集       | 管理员可创建新的规则集，并填写规则集名称、适用说明、创建学年等信息       |
| 复制规则集       | 管理员可基于已有规则集复制生成新规则集，便于在上一学年基础上调整         |
| 编辑规则集       | 管理员可修改规则集中的分类、规则项、分值、上限、材料要求和审核说明       |
| 删除或停用规则集 | 未被使用的规则集可删除；已绑定学年或产生计算结果的规则集应支持停用或归档 |
| 指定学年规则集   | 管理员可为每个学年指定一套规则集，作为该学年综测申报、审核和计分依据     |
| 规则项分类管理   | 管理员可新增、修改、删除规则项分类，并设置分类名称、排序和说明           |
| 类内规则管理     | 管理员可为分类设置统一限制，如最高分、是否允许重复申报、是否取最高项     |
| 规则项管理       | 管理员可配置具体规则项，包括名称、分值、计算方式、材料要求、审核要求等   |
| 团体分配规则管理 | 管理员可设置团队成果中不同成员角色的分数分配方式                         |
| 规则变更提示     | 修改已绑定学年的规则集时，系统应提示该修改可能影响既有综测成绩           |
| 成绩重新计算     | 规则变更后，系统应支持对该学年既有综测成绩进行重新计算                   |
| 变更记录         | 系统应记录规则修改人、修改时间、修改内容和影响范围                       |
|                  |                                                                          |

| 可配置内容           | 示例                                  |

| 分值标准调整         | 国家级一等奖由 5 分调整为 6 分        |
| 规则项分类调整       | 将某些项目归入“科研竞赛类”          |
| 类内分值上限调整     | 科研竞赛类最高加分由 8 分调整为 10 分 |

| 适用学年指定         | 2024-2025 学年使用新版规则集          |

### 8. 规则变更与重算要求

当管理员修改某一学年正在使用的规则集时，系统应进行风险提示，并触发成绩重算流程。
推荐流程如下：

```
管理员修改规则
        ↓
系统判断该规则集是否已绑定学年
        ↓
若已绑定，提示可能影响既有成绩
        ↓
管理员确认修改
        ↓
系统重新计算该学年综测成绩
        ↓
生成新的计算结果，并保留变更记录
```

为保证结果可追溯，系统不应简单覆盖历史结果，而应保留规则变更记录和成绩重算记录。

### 9. 权限要求


| 角色       | 权限                                               |
| ---------- | -------------------------------------------------- |
| 系统管理员 | 可维护全部规则集，指定学年规则集，触发成绩重算     |
| 学院管理员 | 可维护本学院规则集，查看本学院规则变更记录         |
| 审核人员   | 可查看规则项、材料要求和审核说明，不可随意修改规则 |
| 学生       | 可查看与本人申报相关的规则说明和材料要求           |
|            |                                                    |
