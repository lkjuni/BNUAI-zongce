# 规则项唯一申报与审核材料预览

## 1. 本次修改原因

此前同一条学院细则可能有两种表达：既可以用“汇总节点 + 多个规则项”表达，也可以把多个独立成果塞进一个规则项的分级给分配置。两种表达会让申报颗粒度、重复申报和核算方式变得不确定。

本次增加统一定义：**一名学生在一个学年内，对同一个规则项最多只能有一条申报。**

## 2. 节点职责

### 汇总节点 `aggregate`

- 可以包含汇总节点或规则项；
- 配置 `aggregation_type` 和 `max_score`；
- 不接受学生申报；
- 不挂计分配置、申报字段、材料要求、审核要求或团体分配规则；
- 规则发布时不允许只有一个子节点的冗余汇总层。

### 规则项 `item`

- 必须是申报入口；
- 不能包含子节点；
- `aggregation_type` 和 `max_score` 必须为空；
- 基础分只由 `rule_calculation_config` 计算；
- 固定 `allow_repeat = 0`；
- 同一学生、同一学年只能产生一条申报。

分级给分只描述同一成果内部互斥的状态。例如科研项目的项目级别、主持/参与和结题/优秀可以放在一个规则项内。ICPC 和 CCPC 可能同时获奖，因此必须拆成两个规则项，再由“专业竞赛类”汇总。

## 3. 数据库约束

`application_record` 新增稳定字段：

```sql
rule_item_code VARCHAR(100) NOT NULL
```

并建立唯一键：

```sql
UNIQUE (academic_year_id, student_id, rule_item_code)
```

接口会在创建前查询并返回 `409`，唯一键负责拦截并发请求。规则版本变化后节点 ID 可能不同，但稳定规则编码不变，因此仍然无法绕过限制。

## 4. 默认规则重构

规范版默认规则共包含：

- 15 个汇总节点；
- 34 个规则项；
- 0 个只有一个子节点的冗余汇总节点；
- 每个规则项各有计分配置、材料要求和审核要求。

主要变化：

- “项目成果 → 项目成果认定”合并为规则项“科研项目”；
- “论文成果 → 论文成果认定”合并为“年度最高论文成果”；
- ICPC、CCPC、数学建模等不同竞赛拆成平级规则项；
- 挑战杯、互联网+、京彩大创、京师杯等拆成独立规则项；
- 教师素养类不同比赛和证书拆开；
- 文体比赛拆成校级文体比赛和运动会项目；
- 寒暑期社会实践与国家级重大专项活动拆开；
- 文明宿舍、学术实践活动、班级活动、党支部活动、团支部活动拆开。

## 5. 审核材料预览

审核详情现在直接显示 `application_attachment`：

- PDF 使用内嵌阅读器；
- 图片在详情中等比显示；
- 其他文件提供打开链接；
- 同时显示文件名、MIME 类型和材料审核状态。

审核人可以在同一界面查看字段、材料和历史审核记录。

## 6. 验证结果

- 默认树结构验证通过；
- 同一学生重复创建同一规则项申报返回 `409`；
- 单子节点汇总版本发布返回 `409`；
- 增加第二个平级规则项并配置计分后可以发布；
- 学生上传 PNG 后，学委审核详情成功显示文件名、类型、打开链接和图片预览；
- 申报、材料提交、审核及自动核算回归通过。

验证脚本：`scripts/verify-rule-item-constraint.mjs`。
